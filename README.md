## About This Repo

This repository demonstrates a structured SOP library for an internal engineering and operations team at a DevTools company. Each SOP follows a consistent template designed for clarity, repeatability, and ease of maintenance.

These documents are written for a technical audience (engineers, DevOps, and release managers) and are intended to be maintained in a docs-as-code workflow alongside the product codebase.

## SOP Standards Applied

- **Consistent structure** — every SOP uses the same template for predictable navigation
- **Clear ownership** — each SOP has a named owner and review cadence
- **Version control** — all changes tracked in Git with dated changelog entries
- **Actionable steps** — procedures written as numbered steps, not prose narratives
- **Defined scope** — each SOP states explicitly what it covers and what it does not
